# GCP 利用基本方針

## 1. はじめに

このドキュメントは、当社がGoogle Cloud Platform (GCP) を安全かつ効率的に利用するための基本方針を定める。無秩序な利用によるコストの増大やセキュリティリスクを防ぎ、将来のサービス拡大に耐えうる統制の取れた基盤を構築することを目的とする。

## 2. 全体構成

### 2.1. 請求アカウント

* **原則:** 請求アカウントは会社として1つに集約する。
* **実践:** 当社のGCP利用における請求は、経理部が管理する全社統一の請求アカウントに紐付ける。

### 2.2. プロジェクト構成

* **原則:** サービスや環境（開発・ステージング・本番）ごとにプロジェクトを完全に分離し、環境間の相互影響を防止する。
* **実践:** 各サービスは、以下の通り環境ごとにプロジェクトを作成する。以下はWebサービス `my-app` の例である。

| 環境 | プロジェクト名 | プロジェクトID |
| :--- | :--- | :--- |
| 開発 | `my-app-dev` | `my-app-dev-[unique-hash]` |
| ステージング | `my-app-stg` | `my-app-stg-[unique-hash]` |
| 本番 | `my-app-prod` | `my-app-prod-[unique-hash]` |

*(プロジェクトIDは作成時に自動で付与されるユニークな文字列を含みます)*

## 3. 命名規則 (Naming Conventions)

リソースの識別を容易にするため、以下の命名規則を遵守する。

* **GCPプロジェクト:** `[サービス名]-[環境名]` (例: `my-app-dev`)
* **VMインスタンス:** `[サービス名]-[環境名]-[役割]-[連番]` (例: `my-app-prod-web-01`)
* **Cloud Storageバケット:** `[会社プリフィックス]-[サービス名]-[環境名]-[用途]` (例: `acme-my-app-prod-backup`)
    * *(バケット名は全世界でユニークな必要があるため、会社やサービス固有の接頭辞をつけます)*

## 4. コスト管理方針

意図しないコストの発生を防ぎ、利用状況を明確化する。

* **ラベル付けの義務化:** すべてのプロジェクトと主要リソースに、以下のラベルを必ず付与する。
    * `service`: `[サービス名]` （例: `my-app`）
    * `owner`: `[責任者名]`
* **予算アラートの設定:** 各プロジェクトに予算アラートを設定し、コスト超過リスクを早期に検知する。
    * 開発/ステージング: 月額 `¥1,000` を超過した場合に通知
    * 本番: 月額 `¥3,000` を超過した場合に通知
* **定期的な確認:** 各プロジェクト責任者は、毎月第1営業日に「お支払い」レポートを確認し、想定外のコストが発生していないかチェックする。

## 5. 権限管理 (IAM) 方針

「最小権限の原則」に基づき、必要最小限の権限のみを付与する。

* **請求先アカウント管理者:** 経理部およびインフラ管轄部署の指名担当者のみ。
* **プロジェクト オーナー:** 各サービスの開発責任者。担当サービスにおけるインフラの構築・変更に関する権限を持つ。
* **編集者:** 担当サービスの開発担当者。インフラの変更が可能。
* **閲覧者:** 運用監視担当者やその他関係者。リソースの閲覧のみ可能。

## 6. セキュリティ基本方針

基本的なセキュリティ対策を徹底し、会社の資産を保護する。

* **認証:** GCPにアクセスする全Googleアカウントは、**2要素認証 (2FA)** を必須とする。
* **サービスアカウントキーの管理:** サービスアカウントのキーファイル（`.json` ファイル）は、**絶対にGitなどのコードリポジトリに含めないこと。** 厳重に管理し、定期的な棚卸しを実施する。
* **APIキーの制限:** APIキーを作成する場合は、利用可能なAPIやアクセス元IPアドレスを必ず制限する。

## 7. 新規サービス（プロジェクト）追加時のルール

新しいサービスをGCPで開発する際は、必ず以下の手順を踏むものとする。

1.  **プロジェクト作成申請:** インフラ管轄部署へプロジェクト作成を申請する。
2.  **命名規則の遵守:** `[新サービス名]-[環境]` の命名規則に従い、プロジェクトが作成される。
3.  **請求アカウントへの紐付け:** 作成された全プロジェクトは、会社の公式請求アカウントに紐付けられる。
4.  **初期設定の実施:**
    * 必須ラベル (`service`, `owner`) を全プロジェクトに付与する。
    * 予算アラートを設定する。
5.  **権限設定:** IAM方針に基づき、プロジェクト責任者が関係者に適切な役割を割り当てる。
